 18      """OpenVX graph parser class 
 19   
 20      Uses a minidom parser to parse the input file specified by the filename argument. 
 21   
 22      The first thing that the GraphParser does when it is instansiated is to parse the 
 23      XML file provided in the filename argument. 
 24   
 25      Then the GraphParser creates and populates a number of synched lists 
 26      used to store necessary information for the C-code generation 
 27   
 28      Global graph information is generated and stored in the arrays set up in the init function. 
 29      Three main things are needed: 
 30   
 31      1: Lists of all input/output and internal image nodes id's, where the list indices correspond to 
 32      C-array indices for corresponding C-arrays. 
 33   
 34      2: Index-synched lists of all the function nodes and their corresponding OpenVX names 
 35      as well as a node_info object for every function node, containing information 
 36      about all edges (lines) entering or leaving the function nodes and the nodes they connect to. 
 37   
 38      3: Separately index-synced lists for all input and output function nodes and 
 39      lists with the corresponding first input and output indices in the parameter list for 
 40      the node create OpenVX function (as defined in the standard). An input function node 
 41      is a function node that has at least one edge connected to an input image node, 
 42      and similarly for the output function nodes. A given function node can technically be present 
 43      both in the input- and output function node lists though usually this is not the case. 
 44   
 45      After all lists have been created, no mode parsing should be needed, and the 
 46      C-code can be generated and written to file, using queries to the GraphParser methods and 
 47      its generated lists. 
 48   
 49      The GraphParser is mainly intended to provide answers about the graph structure. 
 50      It ideally should not handle the generation of the actual C-code. 
 51      The C-code generating python code uses the Node query methods of the GraphParser 
 52      to make decisions on how to generate the C-code 
 53      based on the global and local graph node information stored in the GraphParser. 
 54   
 55      If the verbose flag is set, the GraphParser will print more detailed information 
 56      to the terminal. 
 57      If the debug_mode flag is set, it will print even more information. 
 58   
 59      If the strip_mode flag is set, the GraphParser will generate C-code without any 
 60      graphmanager or other libs/vision dependencies. 
 61      Such generated code is typically used for standalone OpenVX tests. 
 62      """

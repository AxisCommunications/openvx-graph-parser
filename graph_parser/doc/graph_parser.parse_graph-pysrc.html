  5  Submodules used by parse_graph script 
  6  ------------------------------------- 
  7  The parsing framework uses several files for the generic parsing 
  8  to build up global lists of node dependencies etc. 
  9  For the specific parsing of the individual function nodes, 
 10  parser files are located in the folder node_parse_info 
 11   
 12  The parsing framework makes use of a GraphParser class instance, 
 13  which is defined in the file graphml_parser.py. 
 14  The GraphParser creates a set of global lists of id's for nodes of different fundamental types, 
 15  for instance input image nodes, or function nodes connected to an output node. 
 16  It also handles the graph logic needed to be able to autogenerate the OpenVX compliant C-code. 
 17   
 18  The GraphParser also makes use of the class NodeInfo (defined in graphml_parser.py), which stores 
 19  information about edges connected to a particular node, 
 20  their edge labels and the neighboring nodes connected by these edges. 
 21   
 22  A diagram of how the individual function node parameters are accessed and parsed from this top script can be seen 
 23  in the call diagram below. Note that only ``graph_create_function.parse`` calls the individual node functions even 
 24  though the scripts calls functions from several other submodules in the ``code_generation`` subpackage. 
 25   
 26  .. figure:: ../graph_parser/parse_graph_call_diagram.png 
 27   
 28     Call sequence diagram schematic 
 29   
 30  The script calls in turn the following submodules: 
 31   
 32    1. ``graph_headerfile.py``: Generates a C-header file for the graph. 
 33    2. ``graph_sourcefile_beginning.py``: Generates the beginning of the source file and the graph registration function. 
 34    3. ``graph_create_function.py``: Generates the function that creates the graph skeleton (the graph without input and output images). 
 35       It also calls all the necessary node implementation files in the package ``node_parse_info``.

Source Code for Package graph_parser

 1  """A Python package for autogenerating OpenVX C-code for calculation graphs. 
 2   
 3  Introduction 
 4  ============ 
 5  The main script parse_graph.py parses an OpenVX graph defined in yEd's graphml format, 
 6  to generate C-code for running the graph using the libs-vision GraphManager. 
 7   
 8  See the documentation of the graphml_parser submodule for documentation 
 9  on the required format of the yEd graph in order to be a valid graph 
10  for the graph parser (which nodes, edges and metadata to use etc.). 
11   
12  The parse_graph.py script uses the graph_parser module and its 
13  submodules to parse graph definition files generated by yEd, 
14  which are stored in .graphml format. 
15   
16  Workflow when using the graph parser 
17  ------------------------------------ 
18   
19    1. Graphs are drawn in the graph editor yEd 
20       which saves them in XML format in .graphml files 
21    2. The Python script parse_graph.py is used to parse the .graphml file 
22       and autogenerate source code in C for setting up a "graph skeleton". 
23       The graph skeleton is a complete description of the graph 
24       except that the input and output images are not set 
25       (points to empty images), hence the name "skeleton". 
26    3. The graph_manager object can then use the autogenerated code to 
27       set up I/O images and parameters, and process the graph. 
28   
29  The graph_manager is a C object that is designed to handle 
30  the autogenerated code from the graph parser. 
31  It is designed to be user friendly and hide a lot of tedious 
32  and error-prone details about memory management and handling 
33  of I/O images etc from the user. It expects certain functions 
34  to be generated by the parser. 
35  The graph parser will generate C files named exactly as the yEd .graphml file name, 
36  except the extension .graphml is changed to .c and .h respectively. 
37  One registers a graph for the graph manager by passing a function pointer to a graph registration function. 
38  In order to make the naming easier to remember, the graph parser names the required function 
39  exactly as the C-file name, without the file type extension. 
40  The graph manager expects this function to provide it with a I{create}, and a I{set I/O images} function, 
41  as well as an optional I{set debug images} function. 
42   
43  If the yEd file is named graph_name.graphml, the graph parser will thus generate functions with the following names: 
44   
45    - graph_name\n 
46    - graph_name_create\n 
47    - graph_name_set_io_images\n 
48   
49  and a I{set debug images} function (only generated if any image node is defined as a debug image node): 
50    - graph_name_set_debug_images 
51   
52  If the rules for how to draw the graphs in yEd and how to annotate the graph 
53  with metadata are followed, the generated C code will automatically 
54  get the correct formatting of these functions, as expected by the graph manager. 
55   
56  Note that the OpenVX standard might mention the concept of a graph manager as well. 
57  The graph manager here is a higher level concept that is NOT part 
58  of the OpenVX standard, and the two concepts should not be confused with each other. 
59   
60   
61  Updating the graph parser with new OpenVX calculation nodes 
62  ----------------------------------------------------------- 
63   
64  The first time a new function node is ever being used, the algorithm developer 
65  will need to add some details about the parsing of that function node. 
66  The update process is streamlined to be as minimal and easy to understand 
67  as possible. 
68   
69    1. Add a .py file in the node_parse_info submodule. The file should be 
70       named after the node and contain information about how to parse the node. 
71    2. Add parameter and naming data to the file function_node_library.py 
72       in the node_parse_info submodule. The needed information can be 
73       extracted directly from the OpenVX API specification. 
74   
75  Further overview documentation 
76  ------------------------------ 
77   
78    1. L{The I{parse_graph} module<parse_graph>}: Information about the top level parse graph script that uses the graphml parser framework. 
79    2. L{The I{graphml_parser} package<graphml_parser>}: Information on the metadata the parser expects from the yEd graph definitions. 
80    3. L{The I{node_parse_info} package<node_parse_info>}: Contains further details on how to add new types of nodes to the parser framework. 
81  """
